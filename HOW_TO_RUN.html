1

Install MySQL

The database that stores everything. Install once, runs automatically after that.

1

Download MySQL Installer

Go to mysql.com/downloads/installer → Download the "MySQL Installer for Windows" (the small ~2MB web installer)

2

Run the Installer

Double-click the downloaded file → Choose "Developer Default" → Click Next → Click Execute → Wait for it to finish

3

Set Root Password

During setup it asks for a MySQL Root Password. Type something simple you'll remember:

Recommended root password for development
root123 # Use this exactly — you'll need it in Step 4
Write down your password right now. You cannot connect the backend without it.
✓

Verify MySQL is Running

Open Command Prompt (press Win+R, type cmd, press Enter) and run:

Command Prompt
mysql -u root -p # Type your password when asked # You should see: mysql> # Type "exit" to leave
MySQL Workbench is also installed automatically — it's a visual tool that makes running SQL files much easier.
Terminal — macOS
# Step 1: Install Homebrew (if not already installed) /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # Step 2: Install MySQL brew install mysql # Step 3: Start MySQL now and on boot brew services start mysql # Step 4: Set root password (follow the prompts) mysql_secure_installation # When asked "Set root password?" → Yes → type: root123 # Answer Y to all other questions # Step 5: Test it works mysql -u root -p # Enter password: root123 → you should see mysql>
Terminal — Ubuntu/Debian
sudo apt update sudo apt install mysql-server -y sudo systemctl start mysql sudo systemctl enable mysql # Set up root password sudo mysql # Inside MySQL shell, run these 3 lines: ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'root123'; FLUSH PRIVILEGES; exit; # Now test login mysql -u root -p # Password: root123
2

Load the Database

Run 3 SQL files in MySQL Workbench to create all tables and load sample data.

Recommended method: MySQL Workbench — it's the easiest. It was installed automatically with MySQL on Windows.
1

Open MySQL Workbench

Find it in your Start Menu (Windows) or Applications folder (Mac). Click the connection named "Local instance 3306". Enter your password if asked.

2

Run schema.sql first

In Workbench menu: File → Open SQL Script → browse to your smartcity/database/schema.sql file → click Open → then press Ctrl+Shift+Enter (or click the ⚡ lightning button)

Wait until you see green checkmarks in the Output panel at the bottom. This creates all 9 tables.
3

Run seed_data.sql second

File → Open SQL Script → open database/seed_data.sql → Execute. This loads 100 users + 500 complaints.

4

Run crime_schema.sql third

File → Open SQL Script → open database/crime_schema.sql → Execute. This adds the crime tables and 80 incidents for the map.

✓

Verify it worked

Open a new query tab in Workbench and run this:

MySQL Workbench — New Query Tab
USE smartcity_db; SELECT COUNT(*) AS complaints FROM complaints; -- should say 500 SELECT COUNT(*) AS users FROM users; -- should say 110 SELECT COUNT(*) AS crimes FROM crime_incidents; -- should say 80
If you get an error about stored procedures or triggers, run this first in Workbench: SET GLOBAL log_bin_trust_function_creators = 1; then run the SQL files again.
3

Install Python & Packages

Python runs your Flask backend. You install this once.

1

Check if Python is installed

Command Prompt
python --version # Should say: Python 3.10 or higher # If it says "not recognized" → download from python.org
When installing Python from python.org, check the box "Add Python to PATH" on the first screen before clicking Install.
Terminal
python3 --version # Should say: Python 3.10 or higher
2

Navigate to the backend folder

Command Prompt
cd C:\Users\YourName\Downloads\smartcity\backend # Replace YourName with your actual Windows username # TIP: You can drag the backend folder into the terminal window!
Terminal
cd ~/Downloads/smartcity/backend # Or drag the backend folder into Terminal
3

Create a virtual environment

This keeps Flask packages separate from other Python projects (best practice)

Command Prompt — inside backend folder
# Create the virtual environment python -m venv venv # Activate it venv\Scripts\activate # Your prompt now shows (venv) at the start — like this: (venv) C:\...\backend>
Terminal — inside backend folder
# Create the virtual environment python3 -m venv venv # Activate it source venv/bin/activate # Your prompt now shows (venv) — like this: (venv) user@mac backend %
4

Install all required packages

Terminal (with venv active)
pip install -r requirements.txt # This installs: flask, flask-cors, mysql-connector-python, bcrypt # Takes 1–2 minutes. You'll see packages downloading. # Verify installed correctly: pip list # Should show Flask, mysql-connector-python, bcrypt in the list
4

Connect Flask to MySQL

Tell Flask your MySQL password. This is the most important step.

1

Open app.py in any text editor

Open backend/app.py in Notepad, VS Code, or any editor. Find this block around line 20:

backend/app.py — Find this section
DB_CONFIG = { 'host': 'localhost', 'port': 3306, 'user': 'root', 'password': 'your_password', ← CHANGE THIS LINE 'database': 'smartcity_db', 'charset': 'utf8mb4', }
2

Change the password to yours

backend/app.py — After your change
DB_CONFIG = { 'host': 'localhost', 'port': 3306, 'user': 'root', 'password': 'root123', ← Your MySQL password here 'database': 'smartcity_db', 'charset': 'utf8mb4', }
Use the password you set in Step 1. If you used root123, type exactly that. Save the file after changing.
3

Add crime routes to app.py

Open backend/crime_routes.py → Copy ALL its contents → Paste them into app.py just above the very last line that says if __name__ == '__main__':

This adds the map and crime API endpoints. Without this, the city map won't load live data.
5

Start the Flask Backend

Run app.py and keep this terminal window open the whole time.

Terminal — inside backend folder, venv active
# Make sure you are INSIDE the backend folder # Make sure (venv) is showing in your prompt # Then run: python app.py # You should see output like this: * Serving Flask app 'app' * Debug mode: on * Running on http://127.0.0.1:5000 * Running on http://0.0.0.0:5000
Test it: Open your browser and go to http://localhost:5000/api/health. You should see: {"success": true, "data": {"db": "connected"}}
Important: Do NOT close this terminal window. Keep it running while you use the project. Open a separate terminal for anything else.
Error: "Access denied for user root" → click to fix

Your password in app.py doesn't match your MySQL password.

Fix: re-open app.py and check DB_CONFIG password
'password': 'root123', ← Make this match what you set in Step 1
Error: "ModuleNotFoundError: No module named flask" → click to fix

Either packages aren't installed, or the virtual environment isn't active.

Fix
# Check if (venv) is shown in your prompt. # If not, activate it first: # Windows: venv\Scripts\activate # Mac/Linux: source venv/bin/activate # Then install again: pip install -r requirements.txt
Error: "Port 5000 already in use" → click to fix
Command Prompt
netstat -ano | findstr :5000 taskkill /PID <number from above> /F # Then run python app.py again
Terminal
lsof -i :5000 kill -9 <PID number shown>

Common Errors & Fixes

Click any error to see the fix

Charts not showing / page looks broken
The HTML files load Chart.js and Leaflet from CDN (internet). Make sure you have an internet connection when opening the pages.
Map not loading / shows grey tiles
The map needs internet to load tiles from CartoDB. Check your connection. Also try opening city_map.html via Live Server (port 5500) instead of double-clicking.
"Table doesn't exist" error in Flask

You haven't loaded the SQL files yet, or loaded them in the wrong database.

MySQL Workbench
USE smartcity_db; SHOW TABLES; # If you see fewer than 9 tables, run schema.sql and seed_data.sql again
CORS error in browser console (when connecting backend)
This happens when you open the HTML file directly (file://) instead of through Live Server. Use VS Code Live Server — it serves on http://127.0.0.1:5500 which is whitelisted in Flask's CORS settings.
"python is not recognized" on Windows

Python wasn't added to PATH during installation.

1

Uninstall Python from Control Panel → Programs

2

Re-download from python.org and re-install

3

On the FIRST installer screen, tick "Add Python to PATH" before clicking Install Now
