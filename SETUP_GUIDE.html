Prerequisites — What to Install

You need these 3 things installed on your computer. Nothing else.

  • MySQL 8.0+ — the database server   Download
  • Python 3.10+ — for the Flask backend   Download
  • A browser — Chrome, Firefox, Edge (already have this!)
During Python install on Windows → check "Add Python to PATH" before clicking Install.

Step 1 — Install & Start MySQL

Windows — MySQL Installer

Follow these steps:

  1. Go to mysql.com/downloads/installer
  2. Download "MySQL Installer for Windows" (the ~2MB web installer)
  3. Run it → choose "Developer Default" setup type → Next → Execute
  4. During setup it asks for a root password — set it to something you'll remember (e.g. root123)
  5. Finish installation. MySQL service starts automatically.
You'll also get MySQL Workbench installed — a visual GUI tool that makes everything easier.

Verify MySQL is running

# Open Command Prompt (Win+R → type cmd → Enter)
mysql -u root -p
# Type your root password when asked
# You should see: mysql>

macOS — Using Homebrew (Recommended)

# Install Homebrew first (if not installed)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install MySQL
brew install mysql

# Start MySQL service
brew services start mysql

# Set root password
mysql_secure_installation
# Follow prompts, set password to: root123

Ubuntu / Debian Linux

sudo apt update
sudo apt install mysql-server -y

# Start MySQL
sudo systemctl start mysql
sudo systemctl enable mysql

# Set root password
sudo mysql_secure_installation
# Answer Y to all questions, set password to: root123

# Login to MySQL
sudo mysql -u root -p

Step 2 — Load the Database

This creates all 9 tables, procedures, triggers, views, and loads 500 sample complaints.

Option A — MySQL Workbench (Easiest, Recommended)

  1. Open MySQL Workbench (installed with MySQL)
  2. Click on the Local instance 3306 connection
  3. Go to menu: File → Open SQL Script
  4. Open database/schema.sql → click the ⚡ Execute button (or press Ctrl+Shift+Enter)
  5. Wait for it to finish (you'll see green checkmarks)
  6. Again: File → Open SQL Script → open database/seed_data.sql → Execute
  7. Done! You now have the full database with 500 complaints.
Verify it worked: In Workbench, run this query: SELECT COUNT(*) FROM smartcity_db.complaints; — it should return 500.

Option B — Command Line

# First navigate to your smartcity folder
# Windows example:
cd C:\Users\YourName\Downloads\smartcity

# Run schema (creates tables, triggers, views)
mysql -u root -p < database/schema.sql

# Run seed data (100 users + 500 complaints)
mysql -u root -p smartcity_db < database/seed_data.sql
If you get "mysql is not recognized" on Windows, find MySQL in C:\Program Files\MySQL\MySQL Server 8.0\bin\ and add it to your PATH, or use MySQL Workbench instead.

Verify Database Loaded Correctly

# Login to MySQL
mysql -u root -p

# Run these checks
USE smartcity_db;
SHOW TABLES;
SELECT COUNT(*) FROM complaints;
SELECT COUNT(*) FROM users;
SELECT * FROM vw_dept_performance LIMIT 3;
Expected: 9 tables, 500 complaints, 110 users

Step 3 — Setup Python & Flask Backend

3a. Check Python is Installed

# Check Python version (need 3.10+)
python --version
# OR on some systems:
python3 --version
# You should see: Python 3.10.x or higher

3b. Navigate to Backend Folder

# Open Command Prompt, navigate to backend folder
cd C:\Users\YourName\Downloads\smartcity\backend

# Create a virtual environment (keeps packages isolated)
python -m venv venv

# Activate virtual environment
venv\Scripts\activate
# Your prompt changes to: (venv) C:\...>

# Install all required packages
pip install -r requirements.txt
# This installs: flask, flask-cors, mysql-connector-python, bcrypt
# Navigate to backend folder
cd ~/Downloads/smartcity/backend

# Create virtual environment
python3 -m venv venv

# Activate it
source venv/bin/activate
# Prompt changes to: (venv) user@machine $

# Install packages
pip install -r requirements.txt

3c. Configure Database Password in app.py

Open backend/app.py in any text editor (Notepad, VS Code, etc.) and find this section near the top:

# Find this block around line 20 in app.py:

DB_CONFIG = {
  'host':     'localhost',
  'port':     3306,
  'user':     'root',
  'password': 'your_password', ← CHANGE THIS to your MySQL root password
  'database': 'smartcity_db',
}
Change 'your_password' to whatever password you set for MySQL root in Step 1 (e.g. 'root123'). Save the file.

3d. Start the Flask Server

# Make sure you're in the backend folder with venv active
python app.py

# You should see:
* Running on http://127.0.0.1:5000
* Debug mode: on
Test it: Open your browser and go to http://localhost:5000/api/health — you should see {"success": true, "data": {"db": "connected"}}
Keep this terminal window open! Closing it stops the Flask server. Open a new terminal for other commands.

Step 4 — Run the Frontend

Option A — Just Open in Browser (Simplest)

The frontend has demo data built in. No server needed.

  1. Open your file explorer / Finder
  2. Navigate to the smartcity/frontend/ folder
  3. Double-click index.html
  4. It opens in your browser — done!

Option B — VS Code Live Server (Better, Recommended)

  1. Install VS Code if not already installed
  2. Install the "Live Server" extension (search in Extensions tab)
  3. Open the smartcity/frontend/ folder in VS Code
  4. Right-click index.html → "Open with Live Server"
  5. Browser opens at http://127.0.0.1:5500
Live Server is better because it auto-reloads when you edit files, and CORS works correctly with Flask.

Option C — Python Simple HTTP Server

# Open a NEW terminal, navigate to frontend folder
cd smartcity/frontend

# Start Python HTTP server
python -m http.server 5500

# Open browser at:
http://localhost:5500

Step 5 — Connect Frontend to Backend

To use live MySQL data instead of demo data, make this one change in the HTML files:

# In each HTML file (admin_dashboard.html, etc.), find this line:
const DEMO_MODE = true; ← change to false

# And make sure this matches your Flask port:
const API = 'http://localhost:5000/api';

Common Errors & Fixes

!

❌ "Access denied for user 'root'@'localhost'"

Wrong MySQL password in app.py

# Fix: Open app.py, find DB_CONFIG, change password to match what you set during MySQL install
'password': 'your_actual_password_here',
!

❌ "ModuleNotFoundError: No module named 'flask'"

Packages not installed, or virtual environment not activated

# Make sure venv is activated first, then:
pip install -r requirements.txt
!

❌ "Table 'smartcity_db.complaints' doesn't exist"

Database not loaded yet

# Run the SQL files in MySQL Workbench in this exact order:
1. schema.sql   ← creates the database and tables
2. seed_data.sql ← populates with sample data
!

❌ "CORS error" in browser console

Frontend and backend on different origins

# Use Live Server (port 5500) not file:// protocol
# The Flask app already allows localhost:5500 in CORS settings
!

❌ "Port 5000 is already in use"

# Windows: Kill process on port 5000
netstat -ano | findstr :5000
taskkill /PID <number_shown> /F

# OR change port in app.py last line:
app.run(debug=True, host='0.0.0.0', port=5001)
!

❌ seed_data.sql fails with procedure error

# In MySQL Workbench, before running seed_data.sql, run:
SET GLOBAL log_bin_trust_function_creators = 1;
# This allows stored procedures to work

Quick Run Checklist (Every Time)

After initial setup, here's what to do each time you want to run the project:

  • 1. MySQL is running (starts automatically on Windows/Mac after installation)
  • 2. Open terminal → navigate to smartcity/backend/
  • 3. Activate venv: venv\Scripts\activate (Win) or source venv/bin/activate (Mac/Linux)
  • 4. Start Flask: python app.py — leave this terminal open
  • 5. Open frontend/index.html in browser (or use Live Server)
  • 6. Login with [email protected] / Admin@123 → see the dashboard!
For college demo without backend: Just open index.html → login → navigate to pages/admin_dashboard.html. All charts and analytics display with built-in demo data. No MySQL or Flask needed for the demo!
